# 05. Constraints — 작동 제약

**작성일**: 2026-04-19
**관련 Step**: 인터뷰 Step 6 + 기확정 환경

이 문서는 **설계·구현이 반드시 준수해야 하는 경계**를 정의한다. 요구사항을 "어떻게 푸느냐"에 앞서 "어떤 선 안에서 풀어야 하느냐"를 고정한다.

---

## 5.1 사용자 규모

| 항목 | 값 |
|---|---|
| 동시 사용자 | 1명 (솔로) |
| 계정·권한 체계 | 불필요 |
| 멀티 테넌시 | 미고려 |

→ 단일 사용자 전제 하에 설계를 단순화한다. 인증·권한·공유 기능을 요구사항에서 제거.

---

## 5.2 데이터 규모

| 항목 | 값 |
|---|---|
| 일일 신규 입력 | ~100건 이하 (평균) |
| 장기 보관 기한 | 무기한 |
| 총 데이터 규모 | 초기 수 MB, 1년 후 수백 MB 예상 |

→ 대규모 분산 저장은 불필요. 로컬 단일 파일 기반 저장으로 충분.

---

## 5.3 호스트 환경

| 항목 | 값 |
|---|---|
| 허브 | 상시 가동 기기 1대 (24/7) |
| 클라이언트 | 로컬 작업 기기 1대 + 휴대 기기 |
| 네트워크 | 허브와 클라이언트 간 LAN/VPN 연결 |
| 클라우드 의존 | 외부 연동 서비스에 한해 제한적 허용 |

→ 허브는 **항상 켜져 있다는 전제**에서 설계 가능. 배치·스케줄·감지 작업이 사용자 기기 가동 여부에 독립적.

---

## 5.4 시간·유지보수 예산

| 항목 | 값 |
|---|---|
| 초기 구현 기간 | 2~3개월 (4주 MVP + 이후 확장) |
| 유지보수 인력 | 사용자 본인 (솔로) |
| 주당 유지보수 시간 허용치 | 5시간 이하 (MVP 이후) |

→ **사용자 1명이 운영 가능한 복잡도**가 상한. 설계가 이 한계를 넘으면 재검토.

---

## 5.5 비용 상한

| 항목 | 값 |
|---|---|
| 월 외부 API 비용 | $20 이하 |
| 초기 하드웨어 | 기존 장비 재활용 (추가 구매 없음) |
| 유료 라이선스 | 필요 시 연간 $100 이하 |

→ 오픈소스·로컬 우선. 외부 API는 **품질이 꼭 필요한 경우에만**.

---

## 5.6 언어

| 항목 | 값 |
|---|---|
| 입출력 언어 | 한국어·영어 혼용 |
| 내부 식별자·파일명 | 영어 기반 (kebab-case) |

→ 한·영 혼용 텍스트를 자연스럽게 처리해야 한다. 검색·요약·분류에서 양언어 혼동 방지.

---

## 5.7 개인정보·프라이버시

| 항목 | 값 |
|---|---|
| 기본 방침 | 로컬 우선 처리 |
| 외부 전송 허용 범위 | 품질이 필요한 처리에 한해 제한적 허용 |
| 민감 영역 | 재무·관계·멘탈·건강 데이터 — 로컬 우선 엄격 |
| 로그·캐시 | 로컬에만 저장, 외부 전송 금지 |

→ **기본은 로컬, 예외만 외부.** 도메인별 세부 분리 정책은 v1.0에서 재검토.

---

## 5.8 기존 시스템과의 공존

이 시스템은 **기존에 사용 중인 도구를 대체하지 않는다**. 공존 전제.

| 기존 시스템 | 이 시스템과의 관계 |
|---|---|
| 외부 OKR·KR 트래커 | 독립 운영. 이 시스템은 확정 플랜을 넘기고, 수치 집계·리포트는 트래커가 담당 |
| 외부 태스크 관리 시스템 | 이 시스템이 확정 플랜을 쓰기. 상태·수치는 외부 시스템 권한 |
| 기존 마크다운 메모 | 초기에는 참조만, 이관은 사용자 판단 |
| 캘린더·이메일 등 생산성 도구 | 읽기 연동 (v1.0 이후 범위) |

→ 기존 도구를 보완하는 위치. 경계 침범 금지.

---

## 5.9 응답 지연 허용치

| 작업 | 허용 지연 |
|---|---|
| 입력 캡처·즉시 조회 | 2초 이내 |
| 검색·엔티티 조회 | 2초 이내 |
| 브리핑·회고·세션 생성 | 10~30초 허용 (품질 우선) |
| 외부 시그널 감지 후 반영 | 분 단위 허용 (실시간 불필요) |

→ 빈번한 작업은 빠르게, 깊은 작업은 느려도 품질 우선.

---

## 5.10 장애 대응 제약

| 항목 | 값 |
|---|---|
| 외부 서비스 다운 시 | 로컬 큐에 쌓고 제한된 기능으로 계속 동작 |
| 재시도 정책 | 자동 지수적 백오프 |
| 복구 가시성 | 사용자가 상태 명령으로 대기열·오류 확인 가능 |
| 데이터 손실 허용 | 없음. 입력은 즉시 영속 저장 |

→ 외부 의존이 끊겨도 사용자 흐름이 완전히 멈추면 안 된다.

---

## 5.11 명시적 제외

초기 버전에서 **하지 않을** 것.

- 다중 사용자 공유·협업
- 공개 발행·블로깅
- 모바일 전용 앱
- 음성 녹음 기반 입력 (v1.0 이후 재검토 가능)
- 브라우저 기반 UI
- 실시간 알림·푸시 폭격
- 부재·누락 감지형 nudge
