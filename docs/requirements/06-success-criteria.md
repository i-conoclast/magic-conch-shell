# 06. Success Criteria

**작성일**: 2026-04-19
**관련 Step**: 인터뷰 Step 7

이 시스템이 "성공했다"고 판단할 기준. **시스템 기능 체크가 아닌 사용자 outcome 기준.**

---

## 6.1 6개월 성공 신호 (핵심 3개)

인터뷰 Step 7에서 사용자가 명시적으로 꼽은 신호. 이 셋이 만족되면 프로젝트 성공.

### 1. 플래닝 들이는 시간이 줄어듦
- **대상 문제**: P1 (맥락 재공급 비용), P2 (상위 목표 수동 생성)
- **측정 방식**: 정성 — "플래닝이 힘들지 않다"는 느낌의 지속
- **체감 기준**: OKR·KR 쓰는 시간이 한 달에 몇 시간 줄어듦

### 2. "지난 번 이야기" 반복 설명이 줄어듦
- **대상 문제**: P1 (맥락 재공급)
- **측정 방식**: 정성 — 플래닝·세션에서 "지난번 뭐 했지?" 질문 빈도 감소
- **체감 기준**: 세션 시작 시 자연스럽게 이어받음

### 3. 여러 워크플로가 실제로 로테이션으로 돌아감
- **대상 문제**: P5 (반복 워크플로 구축 비용)
- **측정 방식**: 정량 — 활성 워크플로 수, 실행 빈도
- **체감 기준**: ML 공부·영어·재무 등 최소 3개 워크플로가 주 1회 이상 사용됨

---

## 6.2 우선순위 낮은 신호

인터뷰에서 사용자가 상대적으로 덜 중요하다고 판정.

### (낮은 우선순위) 외부 변화에 플랜 자동 적응
- **대상 문제**: P3
- **체감 기준**: 면접·시장 신호 감지 시 플랜 제안 자동 조정
- **비고**: 요구사항에는 포함되지만 "성공의 직접 지표"는 아님. 6개월 뒤 이게 안 되도 큰 실패는 아님.

---

## 6.3 MVP (4주) 최소 승리 조건

MVP 직후 "올와, 돌아간다"고 느낄 **단 하나의 조건**.

### 🎯 아침 브리핑 하나가 진짜로 돔

- "이 자식 진짜 내를 알아"라는 느낌을 주는 아침 브리핑
- 오늘 해야 할 것 + 어제 놓친 것 + 최근 변화가 한 메시지에 담김
- 사용자가 브리핑을 읽고 답장하며 플랜 확정까지 자연스럽게 이어짐

**이 하나가 되면 MVP 성공.** 나머지 기능(엔티티 자동 태깅·세션 승계·저녁 회고 등)은 보조.

---

## 6.4 MVP 실패 적신호

MVP 직후 "엉성 직전에 포기하게 만들" 경고 신호.

### 🚨 매일 고치는 버그가 매일 나옴
- **왜 위험**: 안정성 부재는 사용 동기를 가장 빠르게 꺾음
- **대응 기준**: 만약 이 상태라면 기능 추가 중단하고 안정화에 집중

### (상대적으로 낮은 우려 — 인터뷰에서 사용자가 크게 안 꼽음)
- 캡처 이탈: 2주 연속 캡처 없어 잊힘 — 습관 설계 문제지만 치명적 아님
- 아침 브리핑 무시: 신뢰 문제지만 튜닝 가능
- 외부 반영 실패: 기술적으로 해결 가능

---

## 6.5 2주 단기 체크포인트

MVP 4주 중 절반 시점에서 점검할 지표.

- [ ] 하루 평균 캡처 3건 이상 유지
- [ ] 아침 브리핑 수신 후 응답률 50% 이상
- [ ] 주 1회 이상 엔티티 관련 검색 실행
- [ ] 크래시 없이 1주 연속 동작

---

## 6.6 장기(6~12개월) 성공 지표

- 시스템 없이 하루를 보내면 **허전하다**고 느낌
- 반복 워크플로(ML·영어·재무 등) 3개 이상이 한 달 내내 사용됨
- 엔티티 프로필 50개 이상 축적 (사람·회사·직무·책 포함)
- 메모리 기록 총량 1000건 이상
- 시스템 유지관리 시간이 주 2시간 이하로 안정화

---

## 6.7 성공 기준의 검증 방법

숫자만 보지 않는다. 사용자 본인의 주관적 경험이 최종 판단 기준.

- **정량 지표**: 캡처 빈도, 세션 실행 횟수, 엔티티 축적 수 등
- **정성 피드백**: 월 1회 간단 회고 — "이 시스템이 도움됐나?", "계속 쓰고 싶나?"
- **행동 변화**: 플래닝 시간 감소, 외부 시스템과의 충돌 없음, 유지 피로 낮음

**공식 실패 선언**: 위 세 축이 모두 부정적이면 프로젝트 종료·재설계.
**공식 성공 선언**: 6.1의 3개 신호가 6개월 지속되면 성공.
