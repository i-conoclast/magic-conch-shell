# G. Evolution — 사용 시나리오

---

## Scenario G1.1: 세션 요약 자동 저장

**Given**: 사용자가 30분짜리 ML 공부 세션을 마쳤다.
**When**: 세션이 정상 종료된다.
**Then**:
- 시스템이 대화 요약을 자동 생성 (오늘 주제·핵심 배운 점·다음 과제)
- 전체 대화 로그와 함께 저장됨
- 저장된 요약은 다음 세션 진도 승계에 활용됨 (FR-E3 연계)
- 사용자가 명시적으로 "저장해줘"를 하지 않음

---

## Scenario G1.2: 저장 실패 복구

**Given**: 세션 종료 시점에 메모리 저장소가 일시 다운.
**When**: 자동 저장이 시도된다.
**Then**:
- 로컬 임시 저장소에 요약·로그 보관
- 저장소 복구 후 자동 재시도
- 데이터 손실 없음

---

## Scenario G2.1: 자동 엔티티 태깅

**Given**: "Jane Smith"와 "Anthropic"이 정식 엔티티로 등록됐다.
**When**: 사용자가 "Anthropic 2차 인터뷰 끝. Jane이 팁 몇 개 줬음"이라고 메모.
**Then**:
- 저장 시 본문에서 "Jane"·"Anthropic" 자동 감지
- 두 엔티티와 메모 간 양방향 연결 생성
- 승인 없이 즉시 적용 (연결은 되돌리기 쉬움)

---

## Scenario G2.2: 신규 고유명사 감지

**Given**: 사용자가 "오늘 Dario 인터뷰 봤음. 인상적이었음"이라고 메모.
**When**: "Dario"가 알려진 엔티티가 아니다.
**Then**:
- 엔티티 초안 자동 생성 (FR-C1 연계)
- 초안이 저녁 회고 승인 인박스에 올라감
- 이 메모는 초안과 연결됨 (초안이 승인되면 연결 유지)

---

## Scenario G3.1: 제안 인박스 침입성 없음

**Given**: 하루 동안 엔티티 초안 3건, 스킬 승격 제안 1건, 중복 후보 1건이 쌓였다.
**When**: 사용자가 낮에 다른 작업에 집중 중이다.
**Then**:
- 어떤 알림도 사용자에게 가지 않음
- 인박스에 조용히 쌓임
- 낮 시간 접점 없음 원칙 준수

---

## Scenario G3.2: 저녁 회고에서 일괄 처리

**Given**: 인박스에 5건이 쌓인 상태.
**When**: 저녁 회고가 시작된다.
**Then**:
- 회고 흐름 중 "오늘 처리 대기 5건"이 제시됨
- 각 항목을 하나씩 대화로 처리 (승인/거절/미루기)
- 처리 완료 후 회고 다음 단계(진행 확인·느낀 점 등)로 이어짐

---

## Scenario G3.3: 미처리 항목 유지

**Given**: 사용자가 저녁 회고 중 2건만 처리하고 나머지 3건은 "내일"로 미뤘다.
**When**: 다음날 저녁 회고.
**Then**:
- 3건이 내일 회고 인박스에 다시 등장
- 처음 쌓인 시점이 표시됨 ("이틀 전부터 대기 중")

---

## Scenario G4.1: 사용자 모델 열람

**Given**: 시스템이 한 달간 사용자 패턴을 학습했다.
**When**: 사용자가 사용자 모델 파일을 연다.
**Then**:
- "자주 쓰는 도메인: 커리어 40%, 재무 25%..."
- "아침 접점: 평균 7:15"
- "거절된 제안 패턴: 주말 학습 세션 제안 3회 거절"
- 사용자가 틀린 항목을 직접 편집·삭제 가능

---

## Scenario G4.2: 사용자 모델에 규칙 추가

**Given**: 사용자가 "주말엔 재무 관련 챌린지 안 받겠다"는 선호가 있다.
**When**: 사용자 모델 파일에 "주말엔 재무 챌린지 비활성화" 규칙을 직접 추가한다.
**Then**:
- 다음 주말부터 해당 규칙이 반영됨
- 재무 관련 챌린지 자동 생략

---

## Scenario G5.1: 논리 비약 챌린지

**Given**: 사용자가 "이번 분기 Anthropic에만 집중 지원"이라고 플랜 확정 중.
**When**: 시스템이 메모리에서 관련 맥락을 확인한다.
**Then**:
- "집중 지원 결정 근거가 이번 달 Anthropic 면접 1건뿐인데, 다른 후보 회사와 비교 없이 확정해도 될까요?" 챌린지
- 사용자가 무시하면 확정 진행
- 사용자가 재검토하면 선택지 확장

---

## Scenario G5.2: 챌린지 비활성화

**Given**: 사용자가 스트레스 받는 주라 챌린지가 귀찮다.
**When**: "챌린지 오늘만 꺼줘" 명령.
**Then**:
- 오늘 하루 논리 비약 지적 기능 비활성화
- 내일 다시 활성화
- 사용자 모델에 기록됨

---

## Scenario G5.3: 다른 실수 유형은 지적 안 함

**Given**: 사용자가 운동을 3일 건너뛰었다.
**When**: 시스템이 이 사실을 알고 있다.
**Then**:
- 시스템은 "운동 건너뛰었다"고 지적하지 않음 (잔소리 범위 밖)
- 사용자가 먼저 물으면 그때 답함
