# H. Integration — 기능 요구사항 (FR)

**카테고리**: 외부 시스템 연동
**관련 Step**: 인터뷰 Step 2 (출력 채널) + Step 3 (외부 시그널 소스)

이 시스템은 **독립 운영하지 않는다**. 기존 사용 중인 외부 시스템과 공존·연동해야 한다.

---

## FR-H1: 외부 태스크 시스템 쓰기

**설명**: 확정된 플랜이 **외부 태스크 관리 시스템**에 자동으로 반영된다. 이 시스템의 메모리와 외부 시스템이 동기 상태를 유지.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 확정된 플랜 1건이 외부 시스템에 태스크 1건으로 등록됨
- 태스크 생성 시 제목·도메인·마감일·관련 KR(있을 경우) 정보 전달
- 이 시스템이 외부 태스크를 **생성**만 담당 (상태 변경·삭제는 외부 시스템 몫)
- 중복 등록 방지 (같은 플랜이 두 번 반영되지 않음)
- 연동 실패 시 로컬 큐에 쌓고 재시도 (FR-I3 연계)

---

## FR-H2: 기존 시스템 읽기 (MVP 유일 외부 시그널)

**설명**: 사용자가 이미 쓰는 **캘린더·태스크 시스템**에서 정보를 **읽기 전용**으로 참조한다. 외부 시그널의 MVP 범위는 이 하나에 한정.

**우선순위**: 높음 (MVP 필수)

**연동 대상 예시**:
- 캘린더 (일정·약속)
- 외부 태스크 시스템 (이 시스템이 쓴 것 + 기존 것 모두)

**완료 조건**:
- 연동된 시스템에서 최근 이벤트·태스크를 분 단위로 가져옴
- 가져온 정보는 아침 브리핑·세션 맥락에 활용됨 (FR-D1 연계)
- 읽기 전용 — 외부 시스템 상태를 바꾸지 않음
- 연동 장애 시 기본 정보 없이도 시스템이 동작 가능

---

## FR-H3: 채팅 앱 양방향 통신

**설명**: 사용자와 시스템 사이 **양방향 채팅**이 가능하다. 아침 브리핑 전달, 플랜 확정 대화, 회고 대화 등이 여기에서 이루어진다.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 시스템이 사용자에게 메시지 전송 가능
- 사용자 메시지가 시스템에 수신됨
- 메시지 송수신 지연이 수 초 이내
- 메시지 이력이 유지되어 맥락 있는 대화 가능
- 채팅 서비스 장애 시 터미널 등 다른 채널로 fallback

---

## FR-H4: 이메일 자동 수집 (v1.0 범위)

**설명**: 지정한 기준에 맞는 이메일을 **자동으로 수집**해 메모리에 저장한다. 취업·채용·중요 공지 등.

**우선순위**: 중간 (**MVP 범위 밖**, v1.0에서 추가)

**완료 조건** (v1.0 기준):
- 사용자가 지정한 필터(보낸 사람·제목 키워드)에 맞는 이메일 감지
- 해당 이메일이 메모리에 자동 저장 (구조화 기록)
- 관련 엔티티 자동 태깅
- 수집된 이메일이 다음 브리핑·세션 맥락에 반영됨

**MVP에서는 해당 사항 없음** — 스펙 정의만 해두고 구현은 v1.0.
