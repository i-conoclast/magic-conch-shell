# C. Entities — 사용 시나리오

---

## Scenario C1.1: 새 면접관 자동 감지

**Given**: 사용자가 면접 후기 템플릿으로 기록을 저장한다. 면접관 이름은 "Jane Smith, ML Recruiter"로 기입됨.
**When**: 저장이 완료된다.
**Then**:
- 시스템이 기존에 없는 이름임을 감지
- "Jane Smith" 초안 엔티티 자동 생성 (타입: 사람)
- 초안에 "ML Recruiter" 역할, 탐지 시점, 첫 언급 기록 링크 자동 기입
- 초안은 저녁 회고 승인 대기열로 올라감
- 공식 검색 결과에는 아직 등장하지 않음

---

## Scenario C1.2: 중복 감지 회피

**Given**: 사용자가 이전에 이미 "Jane Smith" 엔티티를 만든 상태.
**When**: 새 기록에서 또 "Jane Smith" 언급이 있다.
**Then**:
- 새 초안을 만들지 않음
- 기존 엔티티에 새 기록만 연결함

---

## Scenario C2.1: 저녁 회고에서 초안 일괄 승인

**Given**: 하루 동안 쌓인 초안 엔티티 3개가 승인 대기 중이다.
**When**: 저녁 회고 대화를 시작한다.
**Then**:
- 회고 흐름 중 "오늘 감지된 새 인물·회사 3건이 있어요" 안내
- 각 초안마다 최소 정보와 첫 언급 발췌를 보여줌
- 사용자가 하나씩 승인·거절·수정 가능
- 승인된 초안은 정식 엔티티로 이동
- 처리 못한 초안은 내일 회고로 넘김

---

## Scenario C2.2: 승인 과정에서 정보 보완

**Given**: 승인 중인 초안이 "Jane Smith" 한 줄 정보만 있다.
**When**: 사용자가 승인하면서 역할("ML Recruiter"), 소속("Anthropic"), 관계("professional")를 추가 입력한다.
**Then**:
- 정식 엔티티 프로필이 더 풍부한 정보로 생성됨
- 첫 언급 기록에도 해당 엔티티 연결 유지

---

## Scenario C3.1: 기록 본문에 엔티티 언급 자동 연결

**Given**: "Jane Smith" 엔티티가 정식 등록된 상태.
**When**: 사용자가 새 자유 메모에 "Jane한테 follow-up 해야겠다"라고 쓴다.
**Then**:
- 시스템이 본문에서 Jane 언급을 감지
- 해당 메모와 Jane Smith 엔티티 간 양방향 연결 자동 생성
- Jane Smith 프로필 타임라인에 이 메모가 자동 등장

---

## Scenario C3.2: 엔티티 병합 시 참조 갱신

**Given**: 두 개의 중복 엔티티 "Jane Smith"와 "J. Smith"가 존재. 각자 관련 기록이 있다.
**When**: 사용자가 두 엔티티 병합 명령을 내린다.
**Then**:
- 하나의 엔티티로 합쳐짐 (사용자가 지정한 표준 이름)
- 두 엔티티에 연결돼 있던 모든 기록이 통합 엔티티로 재연결
- 이전 개별 엔티티는 삭제됨 (undo 이력은 보존)

---

## Scenario C4.1: 엔티티 프로필 수동 편집

**Given**: 자동 생성된 엔티티 프로필에 "Context" 섹션이 비어 있다.
**When**: 사용자가 프로필을 열어 "멘토 관계, 주 1회 1:1 미팅"이라고 추가한다.
**Then**:
- 해당 내용이 프로필에 저장됨
- 자동 생성 필드와 수동 추가 필드가 구분됨
- 편집 시점이 편집 이력에 기록됨

---

## Scenario C4.2: 외부 편집기 직접 수정

**Given**: 엔티티 프로필이 .md 파일로 저장돼 있다.
**When**: 사용자가 외부 편집기로 파일을 열어 직접 편집한다.
**Then**:
- 파일 변경이 감지됨 (FR-A4 연계)
- 시스템이 변경 내용을 반영하고 관련 참조를 갱신

---

## Scenario C5.1: 동명이인 초안 발생

**Given**: 이미 "Kim" 엔티티가 1명 등록된 상태.
**When**: 새 기록에서 다른 "Kim"이 감지된다.
**Then**:
- 자동 병합하지 않고 새 초안 생성
- 승인 인박스에서 "기존 Kim과 다른 인물인가요?" 옵션 제공
- 사용자가 선택 (별개 생성 · 기존에 병합)

---

## Scenario C5.2: 잘못된 병합 되돌리기

**Given**: 사용자가 실수로 다른 인물 둘을 병합했다.
**When**: undo 명령을 실행한다.
**Then**:
- 병합 전 상태로 복원
- 각 엔티티의 원래 참조가 복원됨
- 복원 과정이 편집 이력에 기록됨
