# F. Voice & Tone — 기능 요구사항 (FR)

**카테고리**: 응답 스타일 (톤 라우팅)
**관련 Step**: 인터뷰 Step 4 (상황별 자동 판단)

시스템이 질문의 성격에 따라 **다른 톤**으로 응답한다. 진지한 질문엔 근거 있는 컨설팅, 가벼운 질문엔 단호한 한 줄, 애매할 땐 혼합. 이 이중성이 장기 사용성을 만든다 (가설 3).

---

## FR-F1: 상황별 톤 자동 판단

**설명**: 사용자 질문이 들어오면 시스템이 **톤을 자동 선택**한다. 사용자가 톤을 명시적으로 지정할 필요 없음.

**우선순위**: 높음 (MVP 필수)

**3가지 톤**:
- **컨설팅 톤**: 진지한 도메인 질문 (커리어·건강·재무·학습·관계 등 결정·분석이 필요한 것)
- **오라클 톤**: 가벼운 질문·사소한 선택·반복 질문 (점심·기분·심심 등)
- **하이브리드**: 가볍지만 관련 기록·근거가 있는 경우 (데이터 한 줄 + 소라고둥 한 줄)

**완료 조건**:
- 질문 성격을 자동 분류 (진지/가벼움/애매)
- 분류 결과에 따라 다른 응답 생성
- 사용자가 톤 모드를 명시적으로 지정할 필요 없음
- 오판 시 사용자가 "진지하게 답해줘" 같은 재요청으로 조정 가능

---

## FR-F2: 컨설팅 톤 (진지)

**설명**: 진지한 질문에 대한 응답은 **선택지 + 각 근거**의 형식을 따른다. 단일 답을 단호하게 내리지 않고, 사용자가 결정할 수 있도록 옵션을 제시한다.

**우선순위**: 높음 (MVP 필수)

**응답 구성**:
- 2~4개의 선택지
- 각 선택지에 왜 이걸 고려해야 하는지 근거
- 근거는 가능한 한 메모리에 있는 기록에서 인용
- 필요 시 시스템의 추천 (하지만 최종 결정은 사용자)

**완료 조건**:
- 진지 질문에 선택지 2~4개 응답
- 각 선택지에 근거 1~2줄
- 근거에 사용자의 과거 기록 참조 포함 (가능할 때)
- 사용자가 결정하지 못해도 "조금 더 생각해볼 수 있도록 추가 질문" 흐름 제공

---

## FR-F3: 오라클 톤 (가벼움)

**설명**: 가벼운 질문에 대한 응답은 **한 줄** 또는 **매우 짧게**. "The shell has spoken" 식 단호함을 톤에 녹임.

**우선순위**: 중간 (MVP 포함)

**응답 사전 예시**:
- "No."
- "Nothing."
- "Maybe someday."
- "The shell has spoken."
- "소라고둥은 답하지 않는다."
- "Try asking again."
- (기타 커스텀 응답 — 운영하면서 확장)

**완료 조건**:
- 가벼운 질문에 한 줄 응답 제공
- 응답 사전이 존재하고 확장 가능
- 사용자가 응답 사전을 편집할 수 있음 (선호·유머 반영)

---

## FR-F4: 동일 오라클 답 연속 방지 (streak 방지)

**설명**: 오라클 응답이 **같은 답 3번 연속**이 되면 다음엔 자동으로 컨설팅 톤으로 전환한다. 진지한 질문인데 계속 장난으로 받는 일 방지.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 최근 N개 응답의 톤·답 기록이 유지됨
- 같은 오라클 답이 3회 연속되면 다음 응답은 컨설팅 또는 하이브리드 톤으로 강제
- streak 카운터 상태를 디버깅 명령으로 조회 가능
- 사용자가 의도적으로 장난을 이어가고 싶으면 설정으로 streak 방지 비활성화 가능
