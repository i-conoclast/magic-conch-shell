# E. Sessions — 완료 조건 (Criteria)

---

## FR-E1: 워크플로 = 마크다운 1파일

- [ ] 워크플로 정의가 단일 .md 파일에 담김
- [ ] 파일에 이름·목적·트리거 조건·세션 흐름·완료 조건이 포함
- [ ] 시스템이 파일을 읽어 세션을 실행할 수 있음
- [ ] 사람이 외부 편집기로 열어 읽고 수정할 수 있음
- [ ] 워크플로 파일은 사용자 접근 가능한 폴더에 저장됨
- [ ] 파일 포맷 명세가 문서로 제공됨 (사용자가 참고용)

---

## FR-E2: 명시적 호출 + 자동 트리거 하이브리드

- [ ] 워크플로 이름으로 명시적 호출 가능 (예: "영어 회화 시작")
- [ ] 명시적 호출 시 지연 없이 해당 세션 진입
- [ ] 사용자 메시지 맥락 분석으로 자동 제안이 발생
- [ ] 자동 제안은 사용자 승인 없이 세션으로 전환되지 않음 (제안만)
- [ ] 사용자가 자동 제안을 거절하면 기본 범용 응답으로 이어짐
- [ ] 세션 중 취소·이탈 명령으로 즉시 벗어날 수 있음
- [ ] 자동 제안의 오탐률이 낮음 (관련 없는데 제안되는 경우 드묾)

---

## FR-E3: 세션 진도 저장·승계

- [ ] 워크플로별로 진도 파일이 독립적으로 존재
- [ ] 세션 정상 종료 시 진도 자동 저장
- [ ] 세션 비정상 종료 시에도 중간 진도 복구 가능
- [ ] 다음 세션 시작 시 진도를 자동 읽어옴
- [ ] 세션 시작 시 "지난번 어디까지 했는지" 요약이 사용자에게 표시됨
- [ ] 사용자가 진도 파일을 직접 열어 편집 가능
- [ ] 진도 초기화 명령 제공

---

## FR-E4: DIY 추가 (사용자 작성 워크플로)

- [ ] 지정된 워크플로 폴더에 새 .md 파일을 추가하면 자동 감지
- [ ] 감지된 워크플로가 즉시 호출 목록에 등장
- [ ] 템플릿 파일이 제공되어 DIY 쓰기 시작 편함
- [ ] 문법 오류가 있는 파일은 해당 워크플로만 비활성화
- [ ] 다른 워크플로는 오류 영향 없이 정상 동작
- [ ] 문법 오류가 있을 때 사용자에게 명확한 오류 메시지

---

## FR-E5: 자동 스킬 승격 (승인 필요)

- [ ] 반복 감지 임계값이 설정 가능 (예: 같은 패턴 N회 이상)
- [ ] 감지된 패턴이 승인 인박스에 제안으로 올라감
- [ ] 제안 내용에 감지된 대화 흐름 요약이 포함됨
- [ ] 사용자가 승인하면 워크플로 파일 초안이 자동 생성됨
- [ ] 초안은 편집 가능하며 명시적 확정 없이 활성화되지 않음
- [ ] 사용자가 거절한 패턴은 다시 제안되지 않음 (또는 더 낮은 빈도로)
- [ ] 승격 제안이 저녁 회고 인박스에 포함됨
