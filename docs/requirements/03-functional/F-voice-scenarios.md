# F. Voice & Tone — 사용 시나리오

---

## Scenario F1.1: 진지 질문 자동 판단

**Given**: 사용자가 "지금 ML 공부 방향 맞아?"라고 질문한다.
**When**: 시스템이 질문을 수신한다.
**Then**:
- 키워드 "ML 공부·방향"이 진지 도메인으로 분류
- 컨설팅 톤(선택지 + 근거)으로 응답 생성

---

## Scenario F1.2: 가벼운 질문 자동 판단

**Given**: 사용자가 "오늘 점심 뭐 먹지"라고 질문한다.
**When**: 시스템이 질문을 수신한다.
**Then**:
- 가벼운 키워드로 분류
- 하지만 메모리에 최근 식사 기록이 있으면 하이브리드 톤 (데이터 근거 + 오라클 한 줄)
- 기록이 없으면 순수 오라클 톤

---

## Scenario F1.3: 톤 재요청

**Given**: 사용자가 "뭐하지"라고 물어봤고 시스템이 오라클 톤으로 "Nothing."이라고 답했다.
**When**: 사용자가 "진지하게 답해줘"라고 재요청한다.
**Then**:
- 같은 질문을 컨설팅 톤으로 재응답
- 오늘 일정·최근 기록 근거로 선택지 제시

---

## Scenario F2.1: 컨설팅 응답 예시

**Given**: 사용자가 "이번 분기 ML 공부에 집중할지, 사이드 프로젝트에 집중할지"라고 묻는다.
**When**: 시스템이 컨설팅 톤으로 응답한다.
**Then**:
- 선택지 A: "ML 공부에 집중" — 이유: 지난 분기 공부 시간 부족, 면접 준비 효과
- 선택지 B: "사이드 프로젝트에 집중" — 이유: 포트폴리오 공백, 지원 시즌 임박
- 선택지 C: "반반 병행" — 이유: 둘 다 미루면 안 되지만 깊이 부족 위험
- 시스템 추천: B (이유 명시)
- 결정은 사용자에게 맡김

## Scenario F3.1: 오라클 응답 — Nothing

**Given**: 사용자가 "뭐 재미있는 거 없어"라고 묻는다.
**When**: 시스템이 오라클 톤으로 응답한다.
**Then**:
- "Nothing."이라는 한 줄 응답만 반환
- 추가 설명 없음

---

## Scenario F3.2: 하이브리드 응답

**Given**: 사용자가 "점심 뭐 먹지"라고 묻고 메모리에 최근 식사 기록이 있다.
**When**: 하이브리드 톤 판정.
**Then**:
- "어제 라면, 그제 초밥. 단백질 부족. — 포케"
- 근거 한 줄 + 오라클 추천 한 단어

---

## Scenario F4.1: 3회 연속 Nothing 후 전환

**Given**: 사용자가 3번 연속 가벼운 질문을 했고 시스템이 "Nothing." 3번 응답.
**When**: 사용자가 네 번째로 비슷한 질문을 한다.
**Then**:
- streak 방지 발동
- 이번엔 컨설팅 또는 하이브리드 톤으로 응답
- 사용자에게 "혹시 정말 뭔가 찾고 있는 거에요?" 같은 후속 질문 가능

---

## Scenario F4.2: streak 방지 비활성화

**Given**: 사용자가 "소라고둥 장난 계속 이어가고 싶음" 설정을 켰다.
**When**: 같은 오라클 답이 5번 연속돼도.
**Then**:
- 강제 전환 발생하지 않음
- 원하는 한 계속 오라클 톤 유지
