# E. Sessions — 사용 시나리오

---

## Scenario E1.1: ML 공부 워크플로 정의 파일

**Given**: 사용자가 ML 공부 세션 파일을 만들고 싶다.
**When**: 제공된 템플릿 기반으로 "ml-study.md" 파일을 작성한다. 내용에 세션 흐름(오늘 주제 확인 → 복습 문제 3개 → 새 주제 학습 → 다음 과제 결정)을 기술한다.
**Then**:
- 시스템이 파일을 감지하고 워크플로 목록에 등록
- 사용자가 "ml 공부 시작"으로 호출할 수 있는 상태

---

## Scenario E2.1: 명시적 호출

**Given**: ML 공부 워크플로가 등록돼 있다.
**When**: 사용자가 "ml 공부 세션 시작"이라고 입력한다.
**Then**:
- 범용 응답 모드에서 해당 워크플로 모드로 즉시 전환
- 지난번 진도 요약이 먼저 표시됨 (FR-E3 연계)
- 워크플로 파일에 정의된 흐름대로 세션 진행

---

## Scenario E2.2: 자동 제안

**Given**: 사용자가 범용 대화 중 "요즘 LoRA 구현 복습 중인데..."라고 말한다.
**When**: 시스템이 메시지 맥락이 ML 공부와 연관됨을 감지한다.
**Then**:
- "ML 공부 세션으로 전환할까요?"라고 제안
- 사용자가 승인하면 워크플로 세션 시작
- 사용자가 거절하면 범용 응답으로 이어짐

---

## Scenario E2.3: 세션 중 이탈

**Given**: 사용자가 영어 회화 세션 중이다.
**When**: 사용자가 "그만"이라고 입력한다.
**Then**:
- 세션이 종료됨
- 현재까지의 진도가 저장됨
- 범용 응답 모드로 돌아옴

---

## Scenario E3.1: 진도 승계

**Given**: 사용자가 지난번 ML 공부에서 "Attention mechanism 복습"까지 진행했다.
**When**: 이번 주 새 ML 공부 세션을 시작한다.
**Then**:
- "지난번: Attention mechanism 복습 완료. 오늘 이어서 Self-attention 실습?"이 먼저 제시됨
- 사용자가 따로 "지난번 뭐 했지?"를 물을 필요 없음

---

## Scenario E3.2: 중단된 세션 복구

**Given**: 사용자가 영어 회화 세션 중간에 네트워크가 끊겨 세션이 강제 종료됐다.
**When**: 다음번 세션을 시작한다.
**Then**:
- 중단된 지점의 진도가 복구됨
- "어제 중간에 끊긴 걸로 보이는데, 이어서 할까요? 아니면 새로 시작할까요?" 선택 제공

---

## Scenario E4.1: DIY 워크플로 추가

**Given**: 사용자가 "재무 리밸런싱 리뷰" 워크플로를 만들고 싶다.
**When**: 템플릿을 복사해 finance-rebalance.md 파일을 작성하고 저장한다.
**Then**:
- 시스템이 파일을 감지하고 즉시 호출 가능 목록에 추가
- 바로 "재무 리밸런싱 시작"으로 호출 가능
- 재시작·관리자 권한 불필요

---

## Scenario E4.2: 문법 오류 격리

**Given**: 사용자가 잘못된 형식으로 워크플로 파일을 작성했다.
**When**: 시스템이 파일을 로드한다.
**Then**:
- 해당 파일 워크플로는 로드 실패로 표시
- 다른 워크플로는 정상 동작
- 사용자에게 "ml-study.md 로드 실패: 트리거 섹션 누락" 같은 명확한 오류 메시지

---

## Scenario E5.1: 반복 패턴 승격 제안

**Given**: 사용자가 매주 "이번 주 지출 얼마였지?" → "카테고리별 정리" → "다음 주 예산 조정" 흐름을 4주 연속 반복했다.
**When**: 저녁 회고 시간이 된다.
**Then**:
- "최근 4주간 이 패턴이 반복됐어요. 워크플로로 저장할까요?" 제안 표시
- 승인하면 "weekly-spending-review.md" 초안이 자동 생성됨
- 초안에는 감지된 흐름이 미리 채워져 있음
- 사용자가 편집·확정하면 다음 주부터 워크플로 모드로 호출 가능

---

## Scenario E5.2: 승격 거절

**Given**: 시스템이 반복 패턴 승격을 제안했다.
**When**: 사용자가 거절한다.
**Then**:
- 해당 패턴이 다시 제안되지 않음 (또는 더 긴 간격 후에만)
- 현재 대화는 범용 응답으로 이어짐
