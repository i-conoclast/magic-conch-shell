# A. Capture — 완료 조건 (Criteria)

각 FR의 **측정 가능한 완료 체크리스트**. 이 항목이 모두 충족되지 않으면 해당 FR은 미완성.

---

## FR-A1: 자유 메모 한 줄 캡처

- [ ] 터미널 입력 경로로 한 줄 캡처 후 2초 이내 "저장됨" 확인 출력
- [ ] 채팅 앱 입력 경로로 한 줄 캡처 후 2초 이내 "저장됨" 확인 응답
- [ ] 마크다운 파일에 쓴 한 줄이 분 단위 이내 시스템에 반영
- [ ] 3개 채널에서 들어온 메모가 동일한 저장 구조로 저장됨 (파일 포맷·필드 일치)
- [ ] 저장 직후 검색 명령으로 해당 메모를 찾을 수 있음
- [ ] 외부 네트워크·서비스 차단 상태에서도 캡처 성공
- [ ] 도메인·엔티티 지정 없이 캡처한 메모도 정상 저장됨

---

## FR-A2: 구조화 기록 캡처

- [ ] 최소 3종 템플릿(면접 후기·회의록·학습 진행)이 기본 제공
- [ ] 템플릿 선택 시 해당 필드 목록이 사용자에게 제시됨
- [ ] 필드를 일부만 채우고 저장해도 실패하지 않음
- [ ] 저장된 구조화 기록은 자유 메모와 구분되는 표시(예: 타입 필드)로 저장됨
- [ ] 사용자가 새 템플릿을 .md 형식으로 추가하면 다음 캡처부터 선택지에 나타남

---

## FR-A3: 도메인·엔티티 선택 캡처

- [ ] 캡처 옵션으로 도메인 1개 지정 가능 (제공 도메인 외 값 입력 시 경고)
- [ ] 캡처 옵션으로 엔티티 N개 지정 가능 (기존 엔티티 목록에서 선택)
- [ ] 새 엔티티 이름을 입력하면 초안(draft) 엔티티로 자동 생성
- [ ] 도메인·엔티티 미지정 메모도 저장·검색 가능
- [ ] 지정된 엔티티와 메모 간 상호 참조가 자동 생성됨

---

## FR-A4: 마크다운 파일 watcher 기반 자동 수집

- [ ] 지정 폴더에 새 .md 파일을 생성하면 1분 이내 시스템이 감지·수집
- [ ] 이미 수집된 파일을 수정하면 변경 내용이 반영됨
- [ ] 수집 대상 폴더를 2개 이상 지정할 수 있음
- [ ] 숨김 파일(`.`로 시작) 및 백업 파일(`~`, `.bak` 등) 제외
- [ ] 파일 내부 메타 블록(프론트매터)이 있으면 파싱해서 도메인·엔티티 정보 활용
- [ ] watcher 프로세스가 비정상 종료되면 로그에 기록 + 자동 재시작 시도

---

## FR-A5: 중복 감지

- [ ] 동일한 파일 경로·같은 내용 해시가 들어오면 두 번째는 skip 처리 + 로그
- [ ] 내용 유사도 95% 이상이면 사용자에게 "중복인가요?" 확인
- [ ] 중복으로 판정된 항목은 원본과 연결(중복 레퍼런스)되어 추후 추적 가능
- [ ] 중복 처리 기록이 상태 점검 명령으로 조회 가능
