# G. Evolution — 기능 요구사항 (FR)

**카테고리**: 상호작용·자동화 (메모리 ↔ 에이전트 진화)
**관련 Step**: 인터뷰 Step 5 (상호작용·진화 모델)

사용자가 강조한 핵심: **"메모리와 에이전트가 서로 상호작용하며 발전한다."** 이 카테고리가 그 진화 메커니즘을 정의한다.

---

## FR-G1: 세션 요약·로그 자동 저장

**설명**: 에이전트와의 모든 상호작용(플래닝·회고·세션·자유 대화)은 종료 시 **요약 + 로그**가 자동으로 메모리에 저장된다. 사용자가 명시적으로 "저장해줘"를 할 필요 없음.

**우선순위**: 높음 (MVP 필수)

**저장 내용**:
- 대화 요약 (주요 결정·결론·다음 할 일)
- 전체 로그 (원본 대화)
- 메타 정보 (시작·종료 시각, 참여 워크플로, 결과물)

**완료 조건**:
- 세션 종료 시 요약이 자동 생성됨
- 저장된 요약·로그는 이후 검색·타임라인에서 조회 가능
- 저장 실패 시 로컬에 임시 저장 후 재시도 (로그 손실 금지)
- 요약은 대화 분량에 비례해 길이 조절 (긴 대화 → 긴 요약)

---

## FR-G2: 자동 엔티티 태깅

**설명**: 기록·대화·세션 결과에서 언급된 엔티티(사람·회사·직무 등)가 **자동으로 태그로 연결**된다. 사용자가 매번 수동 지정할 필요 없음.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 기록 저장 시 본문에서 알려진 엔티티 이름이 자동 감지됨
- 감지된 엔티티와 기록 간 양방향 연결 자동 생성 (FR-C3 연계)
- 신규 고유명사는 엔티티 초안으로 승격 (FR-C1 연계)
- 자동 태깅 정확도가 사용자 기대치 이상 (오탐·누락 모두 낮게)
- 자동 태깅은 승인 불필요 (연결은 되돌리기 쉬움, 엔티티 생성만 승인 필요)

---

## FR-G3: 제안 인박스 (저녁 회고 일괄 처리)

**설명**: 하루 동안 시스템이 만든 **제안·승인 대기 항목**은 별도 알림 없이 **저녁 회고 시간에 일괄** 처리된다.

**우선순위**: 높음 (MVP 필수 — Step 2의 침입성 원칙 구현)

**인박스에 쌓이는 것**:
- 새 엔티티 초안 (FR-C1)
- 스킬 승격 제안 (FR-E5)
- 중복 엔티티 병합 제안 (FR-C5)
- 그 외 proactive 제안 (기록 제안·외부 시그널 연관성 제안 등 — Step 3의 P4 proactive 트리거)

**완료 조건**:
- 인박스가 실시간으로 쌓이되 사용자에게 즉시 알림 X
- 저녁 회고 시작 시 인박스 전체 목록이 제시됨
- 각 항목마다 승인·거절·미루기 옵션 제공
- 회고에서 처리 못한 항목은 다음 회고로 유지
- 인박스 항목이 일정 수 이상 쌓이면 "처리 대기 N건"으로 부드럽게 상기

---

## FR-G4: 사용자 모델 파일 (열람·수정 가능)

**설명**: 시스템이 학습한 **"당신의 패턴·선호"**는 하나의 파일에 기록되며, 사용자가 **언제든 열어 보고 직접 편집**할 수 있다. 블랙박스 금지.

**우선순위**: 중간 (MVP 포함)

**사용자 모델에 기록되는 것**:
- 자주 쓰는 도메인·엔티티
- 거절된 제안 패턴
- 응답 스타일 선호 (톤·길이)
- 생활 리듬 (아침·저녁 시각 등)
- 사용자가 직접 추가한 규칙·주의사항

**완료 조건**:
- 사용자 모델이 단일 파일로 존재
- 사용자가 파일을 외부 편집기로 열어 읽을 수 있음
- 편집 내용이 시스템 동작에 즉시 반영됨 (또는 명시적 재로드)
- 사용자가 자동 학습된 항목을 삭제·무효화할 수 있음
- 사용자가 규칙을 직접 추가할 수 있음 (예: "주말엔 재무 질문 피함")

---

## FR-G5: 논리 비약 지적

**설명**: 사용자의 결정·의견에 **근거가 약하다고 판단되면** 시스템이 부드럽게 지적한다. 잔소리가 아닌 **챌린지 톤**.

**우선순위**: 중간 (MVP 포함)

**지적 범위**:
- **논리 비약만** 대상: 결정 근거가 얕거나 연결이 약한 경우
- 이외(뭉침·과거 모순·런라임 등)는 범위 밖 (잔소리 방지 원칙)

**완료 조건**:
- 플래닝·세션 대화에서 결정·의견이 나올 때 근거 탐지
- 근거가 약하다고 판단되면 "이 결정 뒤 근거가 ~에 비해 약해 보이는데, 어떻게 생각해?" 식 챌린지
- 강요 아님 — 사용자가 무시하고 진행 가능
- 자주 무시하는 챌린지 패턴은 학습되어 빈도 감소
- 사용자가 "챌린지 기능 잠시 꺼줘"로 일시 비활성화 가능
