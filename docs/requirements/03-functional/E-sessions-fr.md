# E. Sessions — 기능 요구사항 (FR)

**카테고리**: 학습·상담 워크플로
**관련 Step**: 인터뷰 Step 4 (하이브리드 구조·DIY·진도 승계) + Step 1 P5 (찍어낼 수 있는 워크플로)

반복되는 상호작용 패턴(ML 공부·영어 회화·재무 리밸런싱·모의 면접 등)을 **재사용 가능한 틀**로 만드는 기능. 새 워크플로를 하나씩 수동 구현하지 않아도 된다.

---

## FR-E1: 워크플로 = 마크다운 1파일

**설명**: 하나의 워크플로(예: "ML 공부 세션")는 **마크다운 파일 1개**로 정의된다. 사람이 읽고 쓸 수 있는 형식.

**우선순위**: 높음 (MVP 필수)

**워크플로 파일에 포함**:
- 워크플로 이름·목적
- 트리거 조건 (언제·어떻게 시작되는지)
- 세션 흐름 설명 (대화 단계·질문 템플릿 등)
- 진도 상태를 어디에 저장할지 지정
- 완료 조건·출력물 정의

**완료 조건**:
- 시스템이 워크플로 파일을 읽어 해당 세션을 실행할 수 있음
- 사람도 파일을 직접 열어 읽고 수정 가능
- 파일은 사용자가 접근 가능한 위치에 저장됨

---

## FR-E2: 명시적 호출 + 자동 트리거 하이브리드

**설명**: 워크플로는 **두 가지 방식**으로 시작될 수 있다: 사용자가 이름으로 명시적 호출하거나, 시스템이 메시지 맥락에서 자동 감지해 제안.

**우선순위**: 높음 (MVP 필수)

**두 호출 방식**:
- **명시적**: "영어 회화 세션 시작" 같은 명령으로 즉시 해당 워크플로 진입
- **자동 제안**: 사용자 메시지가 특정 워크플로와 관련돼 보이면 "영어 회화 세션으로 전환할까요?" 제안

**완료 조건**:
- 명시적 호출 시 지연 없이 세션 진입
- 자동 제안은 사용자 메시지 맥락 분석 후 발동 (오탐 낮게)
- 사용자가 자동 제안을 거절하면 기본 응답으로 이어짐
- 세션 중에도 취소·이탈 명령으로 언제든 벗어날 수 있음

---

## FR-E3: 세션 진도 저장·승계

**설명**: 한 워크플로를 여러 번 반복해도 **지난번 세션 상태가 이어진다**. 사용자가 "지난번 어디까지 했지?"를 따로 말하지 않아도 된다.

**우선순위**: 높음 (MVP 필수)

**진도 정보**:
- 마지막 세션 종료 지점
- 지금까지의 누적 진도 (세션 회차, 커버한 주제, 완료한 과제 등)
- 다음 세션 시작 시 자동으로 읽어와 맥락 제공

**완료 조건**:
- 워크플로별로 진도 파일이 별도로 존재
- 세션 종료 시 진도가 자동 저장됨
- 다음 세션 시작 시 진도를 자동 읽어오기
- 진도 파일은 사용자가 직접 편집·초기화할 수 있음
- 세션이 비정상 종료돼도 중간 진도가 손실되지 않음

---

## FR-E4: DIY 추가 (사용자 작성 워크플로)

**설명**: 사용자가 직접 워크플로 정의 파일을 작성해 추가할 수 있다. 관리자 권한·재시작 없이 즉시 사용 가능.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 사용자가 워크플로 폴더에 새 .md 파일을 쓰면 자동 감지됨
- 감지된 워크플로는 즉시 호출 목록에 등장
- 워크플로 파일 형식 최소 요구사항 문서 제공 (템플릿 포함)
- 파일 문법 오류가 있으면 해당 워크플로만 비활성화되고, 다른 워크플로는 정상 동작

---

## FR-E5: 자동 스킬 승격 (승인 필요)

**설명**: 사용자가 **자주 반복하는 상호작용 패턴**을 시스템이 감지해, "이걸 워크플로 파일로 만들까요?"라고 **제안**한다. 승인되면 자동 생성.

**우선순위**: 중간 (MVP 포함, 간소 버전 가능)

**승격 과정**:
- 시스템이 유사한 대화 흐름이 N회 이상 반복되면 감지
- 승인 인박스에 "이 패턴을 워크플로로 저장할까요?" 제안 추가
- 사용자가 승인하면 워크플로 파일 초안을 생성 (내용은 편집 가능)
- 사용자가 거절하면 해당 패턴은 다시 제안되지 않음

**완료 조건**:
- 자동 감지 임계값(반복 횟수)이 설정 가능
- 승격 제안이 저녁 회고 인박스에 들어감 (FR-D3 연계)
- 자동 생성된 워크플로 파일 초안에 감지된 대화 흐름이 요약됨
- 초안 이후 사용자가 편집·확정하는 단계 필수
