# A. Capture — 사용 시나리오

각 FR의 **Given-When-Then 시나리오**. 실제 사용 상황에서 요구사항이 기능으로 드러나는 방식.

---

## Scenario A1.1: 이동 중 한 줄 메모

**Given**: 사용자가 이동 중이고 채팅 앱으로 시스템에 연결돼 있다.
**When**: "방금 읽은 논문에서 LoRA 적용 방식 인상적이었음" 한 줄을 채팅 앱에 보낸다.
**Then**:
- 2초 이내 "저장됨" 확인 메시지가 돌아온다
- 저장된 메모는 도메인·엔티티 미지정 상태
- 같은 날 저녁 회고에서 "분류 안 된 메모 1건"으로 표시된다

---

## Scenario A1.2: 터미널에서 즉시 캡처

**Given**: 사용자가 로컬 터미널에서 작업 중이다.
**When**: 빠른 아이디어 한 줄을 캡처 명령으로 저장한다.
**Then**:
- 외부 네트워크 없이도 저장됨
- 같은 순간 다른 기기에서 검색하면 해당 메모가 조회됨(동기화 완료 후)

---

## Scenario A2.1: 면접 직후 템플릿 사용

**Given**: 사용자가 방금 원격 면접을 끝냈다.
**When**: 면접 후기 템플릿을 선택하고 "회사·면접관·인상·다음 단계" 필드 중 일부를 채워 저장한다.
**Then**:
- 미기입 필드가 있어도 저장 성공
- 저장된 기록은 타입 "구조화 기록"으로 분류됨
- 면접관 이름을 입력했으면 해당 인물에 대한 엔티티 초안이 자동 생성됨(연계 FR-C1)

---

## Scenario A3.1: 도메인 지정 캡처

**Given**: 사용자가 재무 관련 메모를 쓰려고 한다.
**When**: 캡처 명령에 도메인을 "재무"로 지정한다.
**Then**:
- 메모가 재무 도메인으로 분류되어 저장
- 이후 도메인 필터 검색에서 해당 메모가 정확히 조회됨

---

## Scenario A3.2: 신규 엔티티 즉석 생성

**Given**: 사용자가 처음 만난 사람에 대한 메모를 쓴다.
**When**: 엔티티 지정에 "Jane Smith — ML Recruiter" 를 입력한다.
**Then**:
- 새 엔티티 초안(draft)이 생성됨 (연계 FR-C1)
- 메모와 초안 엔티티 간 상호 연결이 만들어짐
- 엔티티 초안은 저녁 회고 승인 대기열에 올라감

---

## Scenario A4.1: 외부 편집기 파일 자동 수집

**Given**: 사용자가 즐겨 쓰는 노트 앱에서 마크다운 파일로 메모를 쓴다. 이 앱은 지정 폴더에 .md 파일을 생성한다.
**When**: 사용자가 파일을 저장한다.
**Then**:
- 시스템이 1분 이내 파일을 감지
- 파일 내부에 메타 블록이 있으면 해당 정보로 분류 저장
- 메타 블록이 없으면 자유 메모 형태로 저장

---

## Scenario A4.2: 편집기에서 기존 파일 수정

**Given**: 이미 수집된 마크다운 파일이 있다.
**When**: 사용자가 그 파일을 편집·저장한다.
**Then**:
- 시스템이 변경을 감지하고 저장소에 반영
- 이전 버전은 히스토리 기록으로 보존 (선택적)
- 검색 결과가 갱신된 내용으로 갱신됨

---

## Scenario A5.1: 중복 파일 저장

**Given**: 사용자가 같은 내용을 두 번 저장한다(실수로).
**When**: 두 번째 저장 시도가 들어온다.
**Then**:
- 내용 해시가 일치하면 두 번째는 무시됨
- 상태 점검 명령으로 "중복 1건 skip" 로그 조회 가능

---

## Scenario A5.2: 거의 같은 내용

**Given**: 사용자가 며칠 전 메모와 유사한(하지만 약간 다른) 메모를 쓴다.
**When**: 유사도 95% 이상으로 판정된다.
**Then**:
- 저장 전에 "이전 메모(링크 제공)와 비슷한데, 중복으로 처리할까요?" 확인 질문이 뜬다
- 사용자 선택에 따라 저장·병합·취소
