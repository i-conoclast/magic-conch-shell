# G. Evolution — 완료 조건 (Criteria)

---

## FR-G1: 세션 요약·로그 자동 저장

- [ ] 모든 세션 종료 시 자동 요약 생성
- [ ] 전체 대화 로그가 함께 저장됨
- [ ] 메타 정보(시작·종료·워크플로) 기록
- [ ] 저장 실패 시 로컬 임시 저장 + 재시도
- [ ] 요약 길이가 대화 분량에 적절히 비례
- [ ] 저장된 요약·로그가 검색·타임라인에서 조회 가능
- [ ] 사용자가 명시적 저장 명령을 내릴 필요 없음

---

## FR-G2: 자동 엔티티 태깅

- [ ] 기록 본문에서 알려진 엔티티 이름이 자동 감지됨
- [ ] 감지된 엔티티와 기록 간 양방향 연결 자동 생성
- [ ] 신규 고유명사는 엔티티 초안으로 승격 (FR-C1)
- [ ] 자동 태깅은 승인 불필요 (연결은 쉽게 되돌릴 수 있으므로)
- [ ] 태깅 오탐이 드묾 (관련 없는 연결 최소화)
- [ ] 태깅 누락이 드묾 (명시된 엔티티 놓치지 않기)
- [ ] 자동 태깅 동작 여부를 설정으로 끌 수 있음

---

## FR-G3: 제안 인박스 (저녁 회고 일괄 처리)

- [ ] 제안이 실시간으로 인박스에 쌓이지만 사용자에게 즉시 알림 X
- [ ] 저녁 회고 시작 시 인박스 전체 목록 제시
- [ ] 각 항목마다 승인·거절·미루기 옵션
- [ ] 미루기한 항목은 다음 회고로 이동
- [ ] 인박스 항목 수가 임계값 초과 시 아침 브리핑에 부드럽게 상기
- [ ] 사용자가 수동으로 인박스를 열어 처리할 수 있음
- [ ] 처리 이력이 로그로 남음 (언제 무엇을 승인/거절했는지)

---

## FR-G4: 사용자 모델 파일 (열람·수정 가능)

- [ ] 사용자 모델이 단일 .md 파일로 존재
- [ ] 사용자가 외부 편집기로 열어 읽을 수 있음
- [ ] 파일 편집 내용이 시스템에 반영됨 (자동 또는 명시적 재로드)
- [ ] 자동 학습된 항목과 수동 추가 항목이 구분됨
- [ ] 사용자가 자동 학습 항목을 삭제·무효화 가능
- [ ] 사용자가 규칙을 직접 추가할 수 있음
- [ ] 모델 파일에 학습 시점·출처 메타 정보 포함

---

## FR-G5: 논리 비약 지적

- [ ] 플래닝·세션 대화에서 결정·의견의 근거가 탐지됨
- [ ] 근거가 약한 경우만 지적 (논리 비약 범위로 한정)
- [ ] 지적 톤이 챌린지(부드러운 반문)이지 잔소리가 아님
- [ ] 사용자가 지적을 무시하고 진행 가능
- [ ] 자주 무시되는 챌린지 패턴은 학습되어 빈도 감소
- [ ] 사용자가 챌린지 기능을 일시적으로 끌 수 있음
- [ ] 다른 실수 유형(뭉침·과거 모순 등)은 지적하지 않음 (잔소리 방지)
