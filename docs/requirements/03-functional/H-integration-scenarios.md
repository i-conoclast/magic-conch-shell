# H. Integration — 사용 시나리오

---

## Scenario H1.1: 플랜 외부 반영

**Given**: 아침 플랜 대화에서 오늘 태스크 3건이 확정됐다.
**When**: 확정 명령이 처리된다.
**Then**:
- 외부 태스크 시스템에 3건이 자동 생성됨
- 각 태스크에 마감일·도메인 정보 포함
- 사용자에게 "3건 반영 완료" 확인 메시지

---

## Scenario H1.2: 외부 시스템 상태 변경은 외부에서만

**Given**: 외부 태스크 시스템에서 사용자가 직접 태스크 상태를 "완료"로 바꿨다.
**When**: 이 시스템이 다음 연동 시점에 외부 상태를 읽는다.
**Then**:
- 이 시스템은 해당 태스크를 "완료"로 인식
- 하지만 이 시스템이 외부 상태를 바꾸지는 않음
- 저녁 회고에서 "오늘 완료한 태스크 N개"로 반영됨

---

## Scenario H1.3: 외부 시스템 장애 시 큐

**Given**: 외부 태스크 시스템이 다운됐다.
**When**: 사용자가 플랜을 확정한다.
**Then**:
- 확정 플랜은 로컬에 저장됨
- 외부 반영은 로컬 큐에 쌓임
- 시스템이 주기적으로 재시도
- 외부 복구 시 큐가 순차 반영됨
- 사용자 플래닝 흐름은 멈추지 않음

---

## Scenario H2.1: 캘린더 읽기

**Given**: 사용자 캘린더에 오늘 오후 3시 면접 일정이 있다.
**When**: 아침 브리핑이 생성된다.
**Then**:
- 브리핑에 "오후 3시 면접 예정" 포함
- 관련 엔티티(회사·면접관)가 있으면 연결
- 사용자가 직접 입력하지 않아도 자동 반영

---

## Scenario H2.2: 외부 태스크 읽기 통합

**Given**: 이 시스템이 어제 쓴 태스크 3건 + 사용자가 외부 시스템에 직접 쓴 태스크 2건이 있다.
**When**: 아침 브리핑이 오늘 할 일을 구성한다.
**Then**:
- 5건 모두 포함 (출처와 상관없이)
- 이 시스템이 쓴 것과 외부에서 쓴 것이 구분 표시됨 (선택적)

---

## Scenario H3.1: 채팅 양방향

**Given**: 아침 7시 브리핑 시간.
**When**: 시스템이 브리핑 메시지를 채팅 앱으로 보낸다.
**Then**:
- 사용자가 채팅 앱에서 메시지를 받음
- 답장으로 플랜 확정 대화 시작
- 대화 턴이 이어지며 맥락이 유지됨

---

## Scenario H3.2: 채팅 장애 fallback

**Given**: 채팅 서비스가 일시 다운.
**When**: 아침 브리핑 시간이 된다.
**Then**:
- 채팅 전송 실패
- 시스템이 터미널 출력 또는 로컬 알림으로 fallback
- 사용자가 터미널에서 브리핑을 볼 수 있음
- 채팅 복구 시 이후 상호작용은 채팅으로 복귀

## Scenario H4.1: 이메일 자동 수집 (v1.0)

**MVP 범위 밖.** v1.0 구현 시 다음 흐름 예정:

**Given**: 사용자가 "Anthropic 이메일 자동 수집" 규칙을 설정했다.
**When**: 해당 기준의 새 이메일이 도착.
**Then**:
- 시스템이 이메일을 감지해 메모리에 구조화 기록으로 저장
- 관련 엔티티(Anthropic·발신자) 자동 연결
- 다음 브리핑·세션 맥락에 반영됨
