# B. Memory — 기능 요구사항 (FR)

**카테고리**: 기억·검색
**관련 Step**: 인터뷰 Step 3 (보기 단위: 하루 일지 + 엔티티 타임라인)

저장된 기록을 찾아내고 조망하는 경로. 사용자가 과거의 자신과 대화할 수 있게 하는 핵심.

---

## FR-B1: 자유 질의 검색

**설명**: 사용자가 자연어 한 줄로 검색하면 의미상·키워드상 관련 있는 기록이 반환된다.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 사용자가 한 줄 질의를 입력하면 관련도 순서로 결과가 나옴
- 완전 일치하는 단어만이 아니라 **의미가 가까운** 기록도 후보에 포함
- 한국어·영어 혼용 질의에 대응
- 결과에는 기록의 일부 발췌·저장 시점·도메인·연결된 엔티티가 표시됨

---

## FR-B2: 도메인·엔티티 필터 검색

**설명**: 검색 범위를 특정 도메인 또는 엔티티로 제한할 수 있다.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 도메인 1개를 지정해 해당 도메인 기록만 검색 가능
- 엔티티 1개를 지정해 해당 엔티티 관련 기록만 검색 가능
- 도메인과 엔티티 필터를 동시에 사용할 수 있음
- 필터 없이 검색할 때와 결과 포맷이 동일

---

## FR-B3: 하루 단위 뷰 (일지)

**설명**: 특정 날짜를 지정하면 그날에 캡처된 모든 기록이 시간 순서로 나열된다.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 날짜 1개를 입력하면 그날 기록(자유 메모·구조화 기록 모두)이 시간순으로 표시
- "오늘"·"어제"·"지난 금요일" 같은 상대 표현 지원
- 기록 수가 많으면 도메인별로 그룹핑 옵션 제공
- 자동 생성된 세션 요약·브리핑 결과물도 해당 날짜 뷰에 포함

---

## FR-B4: 엔티티 타임라인 뷰

**설명**: 특정 엔티티(사람·회사·직무 등)를 기준으로 **관련된 모든 기록이 시간 역순**으로 나열된다. "지난 달 Jane 관련된 모든 것" 같은 조회.

**우선순위**: 높음 (MVP 필수 — Step 3의 핵심 보기 단위 중 하나)

**완료 조건**:
- 엔티티 1개를 지정하면 해당 엔티티에 연결된 모든 기록이 시간 역순 표시
- 기간 필터(지난 주·지난 달 등) 추가 가능
- 기록 타입별(자유 메모·면접 후기·세션 요약 등) 구분 표시
- 엔티티가 언급만 된 기록과 공식 연결된 기록 모두 포함

---

## FR-B5: 관련도 정렬

**설명**: 검색 결과가 단순 시간 역순이 아니라 **관련도(relevance)** 기준으로 정렬된다. 오래됐지만 핵심적인 기록이 위로 올라올 수 있어야 한다.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 검색 결과는 기본 관련도 순 정렬
- 사용자가 시간 순 정렬로 변경 가능
- 관련도 계산에 의미 유사성과 단어 일치가 모두 반영됨
- 최근 기록에 소폭 가중치가 붙어 동률일 때 최근 것이 먼저
