# I. Admin — 기능 요구사항 (FR)

**카테고리**: 관리·장애 대응
**관련 Step**: 인터뷰 Step 6 (장애 대응·상태 점검)

사용자가 시스템 상태를 **확인·복구**할 수 있게 하는 기능. 솔로 운영 전제에서 필수.

---

## FR-I1: 상태 점검

**설명**: 사용자가 하나의 명령으로 **시스템 전반 상태**를 확인할 수 있다. 어느 부분이 정상인지·문제가 있는지 한 번에 파악.

**우선순위**: 높음 (MVP 필수)

**점검 항목** (대략):
- 메모리 저장소 접근 가능 여부
- 인덱스·검색 엔진 상태
- 외부 연동 시스템(각각)의 연결 상태
- 백그라운드 프로세스(watcher·스케줄러) 동작 여부
- 최근 입력 시점·오류 로그 요약
- 대기 중인 승인 인박스 항목 수
- 로컬 큐(외부 반영 대기) 잔여 수

**완료 조건**:
- 단일 명령으로 상태 확인 가능
- 결과가 한 눈에 보이는 형태 (OK/경고/오류 색상 구분 등)
- 문제 발견 시 간단한 복구 가이드 또는 명령 안내
- 점검 자체가 빠름 (수 초 이내)

---

## FR-I2: 인덱스 재빌드

**설명**: 검색 인덱스·캐시가 꼬였거나 불일치 의심 시, 사용자가 **인덱스를 재빌드**할 수 있다. 메모리 원본은 건드리지 않음.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 사용자 명령으로 인덱스 재빌드 시작
- 재빌드 중에도 기존 검색이 일부 동작 (완전 중단 없음) 또는 명확한 중단 안내
- 재빌드 완료 후 검색 결과가 갱신됨
- 원본 기록(마크다운 파일)은 건드리지 않음
- 재빌드 진행률을 사용자가 확인 가능

---

## FR-I3: 장애 시 로컬 큐 + 제한 기능 지속

**설명**: 외부 시스템 장애 시, 의존적 작업은 **로컬 큐에 쌓고** 독립적 작업(캡처·검색)은 **계속 동작**한다.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 외부 API 호출 실패 시 해당 요청은 로컬 큐에 저장
- 자동 지수적 백오프 재시도
- 재시도 횟수 상한 존재 (무한 재시도 금지)
- 큐 상태(잔여·처리 중·실패)를 상태 명령으로 확인 가능
- 사용자가 수동으로 큐 재실행·삭제 가능
- 외부 의존 없는 기능(로컬 캡처·검색·엔티티 조회)은 장애 영향 받지 않음
