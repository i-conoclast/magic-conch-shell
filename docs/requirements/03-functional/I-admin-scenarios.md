# I. Admin — 사용 시나리오

---

## Scenario I1.1: 아침 상태 점검

**Given**: 사용자가 아침에 시스템 전반이 정상 동작 중인지 확인하고 싶다.
**When**: 상태 점검 명령을 실행한다.
**Then**:
- 수 초 이내 결과 표시
- 메모리 ✅, 인덱스 ✅, 외부 태스크 ✅, 채팅 ⚠️(지연 중), 캘린더 ❌(연결 불가)
- 경고·오류 옆에 복구 가이드 표시
- 승인 인박스 3건 대기, 로컬 큐 0건 등 요약

---

## Scenario I1.2: 문제 조기 감지

**Given**: 백그라운드 watcher가 어제 밤에 비정상 종료됐다.
**When**: 사용자가 상태 점검을 실행한다.
**Then**:
- watcher 상태가 ❌로 표시
- "Watcher 재시작: `mcs autopilot restart`" 같은 복구 명령 안내
- 사용자가 한 번의 명령으로 복구 가능

---

## Scenario I2.1: 인덱스 불일치 의심 시 재빌드

**Given**: 검색 결과가 이상하다. 최근 기록이 검색에 안 나오는 것 같다.
**When**: 사용자가 인덱스 재빌드 명령 실행.
**Then**:
- 재빌드가 시작되고 진행률이 표시됨
- 재빌드 중에도 기존 검색이 일부 동작 (또는 명확한 중단 안내)
- 재빌드 완료 후 최근 기록이 정상 검색됨

---

## Scenario I2.2: 재빌드 실패

**Given**: 재빌드 중 디스크 오류 발생.
**When**: 재빌드가 실패한다.
**Then**:
- 오류 원인 안내 (예: "디스크 공간 부족")
- 기존 인덱스는 보존됨 (재빌드 중 손상되지 않음)
- 사용자가 원인 해결 후 재시도 가능

---

## Scenario I3.1: 외부 시스템 다운 시 로컬 큐

**Given**: 외부 태스크 시스템이 다운.
**When**: 사용자가 플랜 3건 확정.
**Then**:
- 3건이 로컬에 저장됨
- 외부 반영은 큐에 쌓임
- 자동 재시도 백오프 시작 (1초 → 2초 → 4초 → ...)
- 사용자 플래닝 흐름은 정상 완료됨

---

## Scenario I3.2: 외부 복구 후 자동 처리

**Given**: 외부 시스템이 30분 후 복구됨.
**When**: 시스템이 재시도 시점에 성공.
**Then**:
- 큐에 쌓인 3건이 순차 반영됨
- 사용자에게 "대기 중이던 3건 반영 완료" 알림

---

## Scenario I3.3: 재시도 상한 초과

**Given**: 외부 시스템이 장시간 다운 상태.
**When**: 재시도 5회 모두 실패.
**Then**:
- 해당 큐 항목이 "실패" 상태로 표시
- 사용자에게 "반영 실패 1건 — 수동 확인 필요" 알림
- 사용자가 상태 명령으로 상세 확인·수동 재실행 가능

---

## Scenario I3.4: 장애 중에도 캡처·검색 정상

**Given**: 외부 태스크 시스템 + 채팅 앱 모두 다운.
**When**: 사용자가 터미널에서 자유 메모 캡처·검색 시도.
**Then**:
- 캡처 정상 동작 (로컬 저장)
- 검색 정상 동작 (로컬 인덱스 사용)
- 아침 브리핑만 정상 전달 못함 (채팅 다운)
- 핵심 기능은 멈추지 않음
