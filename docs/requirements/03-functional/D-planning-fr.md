# D. Planning — 기능 요구사항 (FR)

**카테고리**: 플래닝 사이클 (아침·저녁·주간)
**관련 Step**: 인터뷰 Step 4 (에이전트 역할) + Step 2 (접점 시간대)

사용자의 하루·주 리듬에 맞춘 플래닝·회고 사이클. **Step 7의 MVP 최소 승리 조건**("아침 브리핑 하나라도 진짜로 돔")이 여기에 걸려 있다.

---

## FR-D1: 아침 브리핑 생성

**설명**: 매일 정해진 아침 시간에 시스템이 **오늘 해야 할 것·어제 놓친 것·최근 변화**를 한 번에 요약해서 사용자에게 전달한다.

**우선순위**: 최상위 (MVP 최소 승리 조건)

**브리핑에 포함되어야 할 것**:
- 오늘 예정된 플랜·태스크 (외부 태스크 시스템에서 읽어옴)
- 어제 완료하지 못한 항목
- 최근 관련된 외부 변화 (연동된 시스템에서 감지된 이벤트)
- 이번 주 우선순위 1~2개
- 오늘을 위한 질문 한 줄 (사용자 응답 유도)

**완료 조건**:
- 사용자가 지정한 아침 시간에 자동 실행
- 브리핑 결과가 채팅 앱을 통해 전달됨
- 브리핑 한 번 생성에 30초 이내
- 브리핑은 대화 가능한 형식 (사용자가 응답하면 플랜 확정 흐름으로 연결)

---

## FR-D2: 대화형 플랜 확정

**설명**: 아침 브리핑에 사용자가 응답하면 **주고받는 대화**를 통해 오늘의 플랜이 확정된다. 일방향 지시가 아니다.

**우선순위**: 높음 (MVP 필수)

**대화 흐름**:
- 시스템이 제안한 항목에 대해 사용자가 추가·삭제·수정 의견을 줌
- 시스템이 의견을 반영해 업데이트된 목록을 다시 보여줌
- 사용자가 "확정" 할 때까지 반복
- 확정 시점에 플랜이 영속 저장되고 외부 태스크 시스템에 반영 (FR-H1 연계)

**완료 조건**:
- 최소 3턴의 대화가 자연스럽게 흐름
- 턴 사이 맥락(이전 턴 내용)이 유지됨
- 확정 명령어가 명시적 (실수로 확정되지 않음)
- 확정된 플랜은 불변 기록으로 저장 (이후 수정 이력 별도)

---

## FR-D3: 저녁 회고 (대화형)

**설명**: 매일 저녁 사용자가 **하루를 돌아보는** 짧은 대화 세션. 회고 내용뿐 아니라 **오늘 쌓인 승인 인박스**도 여기서 처리한다.

**우선순위**: 높음 (MVP 필수)

**회고에 포함**:
- 오늘 확정했던 플랜 대비 실제 진행 상황 확인
- 오늘 생긴 엔티티 초안 목록 승인 (FR-C2 연계)
- 오늘 쌓인 제안 목록 일괄 처리 (FR-G3 연계)
- 간단한 느낀 점·배운 점 기록 (선택)
- 내일을 위한 한 줄 메모 (선택)

**완료 조건**:
- 사용자가 지정한 저녁 시간에 알림 또는 사용자 호출로 시작
- 대화형 흐름 (완료 체크·승인·기록 한 번에)
- 전체 회고 한 번에 5분 이내로 끝낼 수 있는 분량
- 회고 결과가 그날 일지에 자동 저장됨 (FR-B3 연계)

---

## FR-D4: 주간 리뷰

**설명**: 매주 지정된 시점(예: 일요일 저녁)에 **한 주 전체를 돌아보는** 더 깊은 세션. 심화 분석 포함.

**우선순위**: 중간 (MVP 포함, 간소 버전 가능)

**주간 리뷰에 포함**:
- 한 주간 확정된 플랜 대비 실제 실행 요약
- 도메인별 기록 빈도 변화
- 주목할 엔티티 활동 (예: 한 주간 Jane과 3번 연결된 기록)
- 다음 주 우선순위 제안
- 중장기 목표와 현재 상태 갭 체크

**완료 조건**:
- 매주 지정된 시점에 자동 실행 또는 사용자 호출 가능
- 결과물이 장기 보존되는 형식(노트 파일)로 저장됨
- 한 번 실행에 1분 이내
- 사용자가 결과를 보고 추가 대화 가능 (실행은 원타임이지만 후속 대화 연결)

---

## FR-D5: 확정 플랜의 외부 시스템 반영

**설명**: 대화로 확정된 플랜이 **외부 태스크 관리 시스템**에 자동으로 반영된다. 중복 입력 없음.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 확정된 플랜 1건당 외부 시스템에 태스크 1건이 생성됨
- 태스크에는 제목·마감일·도메인·연결된 KR(있을 경우) 정보가 포함
- 외부 시스템 실패 시 로컬 큐에 쌓이고 재시도 (FR-I3 연계)
- 이미 반영된 플랜이 다시 반영되지 않음 (중복 방지)
- 반영된 플랜 수를 상태 명령으로 확인 가능
