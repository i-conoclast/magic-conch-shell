# C. Entities — 기능 요구사항 (FR)

**카테고리**: 1급 시민 개체 관리
**관련 Step**: 인터뷰 Step 3 (엔티티 축) + Step 5 (자동 태깅·승인)

반복 등장하는 개체(사람·회사·직무·책 등)를 흘러가는 언급이 아닌 **독립된 프로필**로 승격. 기록 간 연결 허브가 된다.

---

## FR-C1: 자동 엔티티 초안 생성

**설명**: 기록에서 새 고유명사(특히 사람·회사·직무·책)가 감지되면 시스템이 **초안 엔티티**를 자동으로 만든다. 사용자의 명시적 요청 없이.

**우선순위**: 높음 (MVP 필수)

**초안의 조건**:
- 정식 엔티티와 분리된 "초안(draft)" 상태
- 사용자 승인 전에는 검색·타임라인 뷰에 공식적으로 등장하지 않음
- 승인 대기열에 올라감 (저녁 회고에서 일괄 처리)

**완료 조건**:
- 새 기록에서 감지된 고유명사가 기존 엔티티에 없으면 초안 생성
- 초안에는 최소 정보(이름·탐지 시점·첫 언급 기록) 자동 기입
- 같은 고유명사가 이미 초안으로 있으면 중복 생성하지 않음
- 감지 정확도가 낮은 경우(문법 오인 등) 초안을 덜 공격적으로 생성

---

## FR-C2: 엔티티 승인·거절

**설명**: 초안 엔티티를 사용자가 저녁 회고 시간에 일괄로 검토하여 **승인·거절·수정**할 수 있다.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 저녁 회고 시간에 대기 중인 초안 목록이 표시됨
- 각 초안마다 감지된 정보와 "첫 언급 기록" 발췌가 보임
- 사용자가 승인하면 정식 엔티티로 이동, 거절하면 삭제
- 승인 과정에서 역할·소속 등 추가 정보를 넣을 수 있음
- 회고 시간에 처리 못한 초안은 다음 회고까지 유지

---

## FR-C3: 자동 양방향 연결 (back-link)

**설명**: 기록과 엔티티 사이에 **양방향 참조**가 자동으로 생성·유지된다. 기록에서 엔티티로, 엔티티에서 기록으로 모두 따라갈 수 있다.

**우선순위**: 높음 (MVP 필수)

**완료 조건**:
- 기록에서 엔티티를 지정하면 엔티티 프로필에 해당 기록이 자동 등장
- 반대 방향(엔티티 프로필에서 관련 기록 목록)도 자동
- 기록 내용에서 엔티티 이름이 본문 언급되면 자동 연결
- 연결된 엔티티가 삭제되거나 병합되면 참조가 자동 갱신됨

---

## FR-C4: 엔티티 프로필 조회·수정

**설명**: 엔티티 프로필은 독립 문서로 존재하며 사용자가 직접 **조회·편집**할 수 있다. 자동 생성된 정보를 수동으로 보완 가능.

**우선순위**: 중간 (MVP 포함)

**프로필 포함 정보**:
- 기본 정보 (이름·역할·소속·첫 접촉일·마지막 접촉일)
- 관련 기록 목록 (타임라인)
- 사용자가 추가한 자유 메모 영역 ("Context", "Next actions" 등)

**완료 조건**:
- 엔티티 지정으로 프로필 열람 가능
- 프로필 편집 가능 (필드 추가·수정)
- 수동 편집한 내용이 자동 정보와 구분되어 보존됨
- 편집 후 저장 시점이 기록됨

---

## FR-C5: 동명이인·중복 처리

**설명**: 같은 이름을 가진 서로 다른 엔티티가 생길 수 있다. 이를 구분·병합할 수 있어야 한다.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 같은 이름이 감지되면 기존 엔티티와 신규 초안을 모두 유지 (자동 병합하지 않음)
- 사용자가 병합·분리 명령으로 정리할 수 있음
- 병합 시 두 엔티티의 참조가 하나로 모아짐
- 잘못된 병합은 되돌릴 수 있음 (undo 지원)
- 식별 정보(역할·소속)로 추정한 중복 후보가 있으면 승인 인박스에서 제안됨
