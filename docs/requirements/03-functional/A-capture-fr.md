# A. Capture — 기능 요구사항 (FR)

**카테고리**: 입력·캡처
**관련 Step**: 인터뷰 Step 3 (메모리 단위 3축) + Step 2 (입력 수단 3종)

시스템이 사용자의 입력을 받아 저장하는 경로. 사용자가 의식적으로 쓰는 메모부터, 파일·채널에서 자동 수집되는 입력까지 포함.

---

## FR-A1: 자유 메모 한 줄 캡처

**설명**: 사용자가 생각·아이디어·후기·관찰 한 줄을 즉시 기록할 수 있다. 입력 직후 영속 저장되고 검색 대상에 포함된다.

**우선순위**: 높음 (MVP 필수)

**입력 채널**:
- 터미널 명령
- 채팅 앱 메시지
- 마크다운 파일에 직접 타이핑

**완료 조건**:
- 사용자가 한 줄 텍스트를 입력하면 2초 이내 저장 확인
- 3개 채널 어디서 들어와도 동일한 저장 위치·형식
- 저장된 메모는 즉시 검색 결과에 노출
- 네트워크·외부 서비스 없이도 저장 성공 (로컬 우선)
- 입력 자체는 어떤 사전 분류·태그 지정 없이도 가능

---

## FR-A2: 구조화 기록 캡처

**설명**: 면접·회의·한 주 실험 진행 같이 반복되는 패턴의 기록을 **템플릿 기반**으로 작성할 수 있다. 자유 메모와 구분되어 저장된다.

**우선순위**: 높음 (MVP 필수)

**지원 템플릿 예시** (초기):
- 면접 후기
- 회의록
- 실험·학습 진행 로그

**완료 조건**:
- 템플릿 종류를 선택하면 필요한 필드(예: 참석자·일시·핵심 결론)가 제시됨
- 필드를 비우고도 저장 가능 (강제 입력 없음)
- 저장된 구조화 기록은 자유 메모와 구분되는 형식으로 표시
- 새 템플릿을 사용자가 추가할 수 있다 (구체 방식은 E 카테고리와 연계)

---

## FR-A3: 도메인·엔티티 선택 캡처

**설명**: 캡처 시점에 관련 **도메인**(5개 라이프 영역 + 학습·일반)이나 **엔티티**(사람·회사·직무·책 등)를 지정할 수 있다.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 캡처 시 옵션으로 도메인 1개를 선택할 수 있음
- 캡처 시 옵션으로 엔티티 1개 이상을 선택하거나 새로 만들 수 있음
- 도메인·엔티티를 지정하지 않아도 캡처 가능 (미분류로 저장)
- 지정된 도메인·엔티티는 검색·필터·타임라인 뷰에서 활용됨

---

## FR-A4: 마크다운 파일 watcher 기반 자동 수집

**설명**: 사용자가 별도 편집기·노트 앱에서 쓴 마크다운 파일이 지정 폴더에 저장되면, 시스템이 **자동으로 감지해 수집**한다. 별도 명령 불필요.

**우선순위**: 높음 (MVP 필수 — Step 2에서 선택된 주요 입력 수단)

**완료 조건**:
- 지정된 폴더에 새 .md 파일이 저장되면 분 단위 이내에 감지
- 파일 내용 변경(수정)도 감지·반영
- 감지된 파일은 자유 메모·구조화 기록 중 적절한 형식으로 분류 저장
- 수집 대상 폴더는 사용자가 여러 개 지정 가능
- 수집 제외 파일(예: 숨김 파일·임시 파일)은 스킵

---

## FR-A5: 중복 감지

**설명**: 동일 또는 거의 동일한 내용이 중복 저장되지 않는다. 특히 파일 watcher와 수동 입력이 겹칠 때.

**우선순위**: 중간 (MVP 포함)

**완료 조건**:
- 동일한 원본(같은 파일 경로·같은 내용 해시)이 두 번 들어오면 두 번째는 무시하거나 업데이트로 처리
- 내용이 95% 이상 유사한 경우 사용자에게 확인 제안 (중복 여부)
- 중복 처리 결과는 로그로 남음
