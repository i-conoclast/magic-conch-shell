# 04. Non-Functional Requirements

**작성일**: 2026-04-19
**관련 Step**: 인터뷰 Step 6

시스템이 **어떤 방식으로** 동작해야 하는지. "무엇을 하는가"(FR)와 달리 "어떻게 하는가"의 품질 경계.

---

## NFR-01: Privacy (프라이버시)

| 항목 | 요구사항 |
|---|---|
| 기본 방침 | 로컬 우선 처리 — 모든 데이터는 로컬 저장소가 원본 |
| 외부 전송 허용 | 품질이 꼭 필요한 처리(진지 응답 등)에 한해 제한적 허용 |
| 민감 영역 | 재무·관계·멘탈·건강 영역은 **로컬 우선 원칙 엄격 적용** |
| 로그·캐시 | 로컬에만 저장, 외부 전송 금지 |
| 사용자 개인 식별 정보 | 외부 전송 시 최소화·익명화 |

**검증 기준**:
- 외부 API 호출 횟수·내용을 사용자가 조회할 수 있어야 함
- 민감 도메인 데이터가 외부로 나가려 할 때 사용자 확인 단계 제공
- 로그 파일이 로컬에만 저장되는지 확인 가능

## NFR-02: Latency (응답 지연)

| 작업 유형 | 허용 지연 |
|---|---|
| 캡처 저장·즉시 조회 | 2초 이내 |
| 검색·엔티티 조회 | 2초 이내 |
| 상태 점검 | 수 초 이내 |
| 브리핑·회고·세션 생성 | 10~30초 허용 |
| 주간 리뷰 생성 | 1분 이내 |
| 외부 시그널 감지 반영 | 분 단위 허용 |
| 인덱스 재빌드 | 사용자가 기다릴 수 있는 시간 (진행률 표시 필수) |

**검증 기준**:
- 반복 작업은 빠르게, 깊은 작업은 품질 우선
- 2초·30초 기준이 정량 측정 가능해야 함

## NFR-03: Availability (가용성)

| 항목 | 요구사항 |
|---|---|
| 허브 가동 | 24/7 |
| 계획 중단 | 설정·업데이트 시 최대 10분 |
| 비계획 중단 | 월 합계 1시간 이내 목표 |
| 재시작 후 자동 복구 | 백그라운드 프로세스 자동 재시작 |

**검증 기준**:
- 허브 재부팅 후 수동 개입 없이 모든 백그라운드 프로세스가 복구됨
- 가용성 지표를 상태 명령으로 조회 가능

---

## NFR-04: Reliability (신뢰성)

| 항목 | 요구사항 |
|---|---|
| 데이터 손실 허용 | 없음 — 입력은 즉시 영속 저장 |
| 외부 장애 대응 | 로컬 큐 + 자동 재시도 (FR-I3) |
| 부분 기능 장애 | 독립 기능은 영향 없이 동작 |
| 재시도 상한 | 5회 (이후 실패 상태로 표시) |

**검증 기준**:
- 외부 장애 모의 테스트 중 로컬 캡처·검색이 정상 동작
- 큐에 쌓인 항목이 복구 후 정상 반영

---

## NFR-05: Portability (이식성)

| 항목 | 요구사항 |
|---|---|
| 데이터 이식성 | 메모리 폴더 하나만 복사하면 다른 기기에서 복원 가능 |
| 포맷 독립성 | 원본 기록이 평문 마크다운 — 전용 포맷 의존 없음 |
| 백업 | 외부 저장소로 주기 백업 지원 |

**검증 기준**:
- 메모리 폴더를 다른 기기로 옮겨 읽을 수 있음
- 외부 편집기로 .md 파일을 열어 내용 확인 가능

---

## NFR-06: Tone & UX (톤·사용자 경험)

| 항목 | 요구사항 |
|---|---|
| 3 Beliefs 구현 | "Planning is a conversation" / "Past you, present you..." / "Answer with data, or answer with nothing" |
| 아첨 금지 | "좋은 질문입니다" 같은 공허한 공손 표현 금지 |
| 단호함 | 진지한 질문에 데이터 근거 있는 단호한 선택지 제공 |
| 유머 허용 | 가벼운 질문에 오라클 톤 (소라고둥 스타일) |
| 잔소리 금지 | 부재·누락 감지형 nudge 금지 |
| 언어 혼용 | 한국어·영어 혼용 자연스럽게 처리 |

**검증 기준**:
- 진지 도메인 질문에 컨설팅 톤 응답 확인
- 가벼운 질문에 한 줄 오라클 응답 확인
- 아첨 표현 탐지 시 수정

---

## NFR-07: Maintainability (유지보수성)

| 항목 | 요구사항 |
|---|---|
| 운영 인력 | 솔로 (사용자 본인) |
| 주당 유지보수 시간 | 5시간 이하 (MVP 이후) |
| 설정 방식 | 파일 기반 — 외부 편집기로 열어 수정 가능 |
| 업그레이드 | 데이터 손실 없는 마이그레이션 보장 |

**검증 기준**:
- 신규 기능 추가 시 기존 데이터가 자동 마이그레이션됨
- 설정 변경이 재시작 없이 반영되거나 명시적 재시작 안내

## NFR-08: Cost (비용)

| 항목 | 요구사항 |
|---|---|
| 월 외부 API 비용 | $20 이하 |
| 하드웨어 | 기존 장비 재활용 (추가 구매 없음) |
| 유료 라이선스 | 연간 $100 이하 |

**검증 기준**:
- 월 비용 추적 가능 (외부 API 사용 로그)
- 비용 상한 근접 시 사용자 경고

---

## NFR-09: Security (보안)

| 항목 | 요구사항 |
|---|---|
| 외부 시스템 인증 정보 | 암호화 저장 (평문 금지) |
| 로컬 데이터 권한 | 사용자 본인만 접근 (OS 권한 활용) |
| 로그 민감 정보 | 자동 마스킹 (토큰·비밀번호 등) |

**검증 기준**:
- 설정 파일에서 평문 토큰이 발견되지 않음
- 로그 출력에서 민감 정보 마스킹 확인

---

## NFR-10: Accessibility (접근성)

| 항목 | 요구사항 |
|---|---|
| 터미널 사용자 | 기본 지원 (CLI 우선 설계) |
| 채팅 앱 사용자 | 기본 지원 |
| 데스크톱 GUI | MVP 범위 밖 |
| 모바일 앱 | MVP 범위 밖 |
| 장애 접근성 | MVP 범위 밖, v2.0 이후 고려 |

**검증 기준**:
- 터미널·채팅 앱 환경에서 모든 핵심 기능 사용 가능
