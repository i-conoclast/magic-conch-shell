# 01. Background — 왜 이 시스템이 필요한가

**작성일**: 2026-04-19
**관련 Step**: 인터뷰 Step 1 (문제 정의)

---

## 1.1 현재 상황

사용자는 이미 여러 생산성·지식 관리 도구를 조합해서 사용 중이다.

- **OKR·KR 트래커**: 5개 라이프 영역(커리어·건강·멘탈·관계·재무)의 목표를 추적하고 6개 외부 데이터 소스에서 활동을 자동 수집하는 시스템을 운영 중
- **자유 마크다운 메모리**: 에이전트가 플래닝할 때 참조하는 개인 맥락 문서를 수작업으로 작성·유지
- **외부 태스크 관리 시스템**: 확정된 할 일과 진행 상황은 외부 시스템에 영속 저장

이 조합은 **정량 추적**(KR·진도)과 **수동 맥락 주입**에는 작동하지만, 아래 영역에서는 공백이 있다.

---

## 1.2 공백 영역 (시스템이 커버하지 못하는 것)

- 자유로운 생각·아이디어·후기의 **구조적 축적**
- 사람·회사·직무 같은 반복 등장 개체의 **프로필화**
- 외부 환경 변화(면접 결과·채용 흐름·시장 흐름)에 따른 **플랜 자동 재조정**
- 반복 워크플로(학습·상담·리뷰 세션)의 **재사용 가능한 틀**
- 대화형 플래닝(일방향 지시가 아닌 주고받기)

이 공백을 기존 도구에 억지로 밀어넣으려 하면 수동 노동이 급증한다.

---

## 1.3 문제 정의 (P1~P5)

인터뷰를 통해 드러난 **다섯 가지 반복되는 통증 지점**.

### P1. 맥락 재공급 비용

**증상**: 계획을 세울 때마다 동일한 맥락(최근 상황·결정 근거·관련 인물)을 매번 다시 공급해야 한다.
**결과**: 플래닝 1회당 실제 의사결정보다 맥락 재설명에 더 많은 시간이 든다.

### P2. 상위 목표·액션 아이템 수동 생성

**증상**: OKR·KR을 처음부터 직접 쓰고, 그것을 일일 태스크로 쪼개는 일도 수동이다.
**결과**: 분기·월 주기마다 큰 시간 블록이 "기획 작업"에 묶인다. 기획하는 시간 때문에 실행할 시간이 줄어든다.

### P3. 외부 환경 변화 무반응

**증상**: 면접 결과·채용 공고 흐름·시장·포트폴리오 리밸런싱 신호 등 **외부 환경**이 변해도 개인 플랜은 자동으로 갱신되지 않는다.
**결과**: 플랜이 외부 현실과 시간차를 두고 어긋난다. 이를 수동으로 맞추는 것은 또 다른 노동.

### P4. 메모리 수동 큐레이션

**증상**: 메모리가 자동으로 축적되지 않는다. 사용자가 "이건 기록해야지" 하고 직접 써야 남는다. 에이전트가 먼저 "이걸 기록해둘까?" 제안하는 구조도 없다.
**결과**: 중요해 보이지 않는 순간들(대화 한 줄·흘러가는 아이디어·작은 결정)은 대부분 사라진다. 나중에 필요할 때 "기록해둘걸" 후회.

### P5. 반복 워크플로 구축 비용

**증상**: 학습 코칭(ML 공부)·재무 리밸런싱 상담·모의 면접·영어 회화 같이 **반복되는 상호작용 패턴**을 agent-interactive하게 만들고 싶지만, 각각을 따로 기획·구현하는 비용이 높다.
**결과**: 하나의 워크플로(예: ML 공부)만 만들어놓고 나머지는 생각만 한 채 방치된다.

---

## 1.4 메타 문제

위 다섯 문제는 표면 증상이고, **공통 뿌리**는 하나다.

> **시스템이 자기 자신을 관리하지 못하기 때문에, 사용자가 유지관리 노동에 시간을 뺏긴다. 그래서 지속이 어렵다.**

- 맥락은 **자동 축적**되지 않아서 매번 공급해야 한다 (P1·P4)
- 목표는 **자동 재구조화**되지 않아서 매번 다시 쓴다 (P2·P3)
- 워크플로는 **찍어낼 수 있는 틀**이 없어서 하나씩 짓는다 (P5)

각 증상을 개별로 해결하는 대신, 이 메타 문제를 공격해야 한다.

---

## 1.5 가설

해결 방향은 세 가지 가설로 좁혀진다. 기술 선택이 아닌 **작동 방식에 대한 가정**.

### 가설 1. 통합 메모리 + 대화형 플래너가 있으면, 플래닝 시간이 줄고 지속이 쉬워진다

개인 맥락을 한 곳에 축적하고, 플래닝이 일방향 지시가 아니라 에이전트와의 대화로 수렴할 때, 매일 반복하기 쉬워진다.

### 가설 2. 반복 등장 개체(사람·회사·직무 등)를 1급 시민으로 다루면, 관계·커리어 추적이 자동화된다

흘러가는 언급을 엔티티 파일로 승격하고 상호 연결하면, 사용자가 별도로 사람 리스트·회사 리스트를 관리하지 않아도 된다.

### 가설 3. 가벼운 질문과 진지한 질문에 **다른 톤**으로 반응하는 이중성이 장기 사용성을 높인다

모든 질문에 똑같이 진지한 시스템은 피곤해진다. 반대로 모든 질문을 장난스럽게 받는 시스템은 신뢰를 못 준다. 상황에 따라 단호한 한 줄과 근거 있는 선택지를 오가는 대화가 장기 관계를 만든다.
