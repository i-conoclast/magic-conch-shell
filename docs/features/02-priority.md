## v1.0 Cut Line (P1) — 8개

MVP 이후 2~3개월 내 추가. 안정화 후 기능 확장.

### P1 엔티티
- **F-24 엔티티 병합·분리**
  - **근거**: 초기엔 동명이인 빈도 낮음. 누적되어야 필요. MVP는 drafts 워크플로로 우선 대응.

### P1 세션
- **F-43 자동 스킬 승격**
  - **근거**: 반복 패턴 감지에는 데이터가 필요. MVP 4주로는 부족. v1.0에서 구현.

### P1 톤
- **F-53 Streak 방지**
  - **근거**: 초기엔 오라클 응답이 적어 streak 발생 드물다. 누적되면 필요.

### P1 진화
- **F-62 Proactive 제안 인박스**
  - **근거**: "이거 기록할까?" 제안은 학습된 패턴 기반. MVP는 사용자 주도, v1.0에서 제안 도입.
- **F-63 사용자 모델 열람·편집**
  - **근거**: MVP 기간 중 학습 데이터 부족. v1.0 시점에 의미 있는 모델 축적.
- **F-64 논리 비약 챌린지**
  - **근거**: 기본 응답 안정화 후 튜닝. 섣불리 챌린지하면 오히려 UX 악화.

### P1 연동
- **F-72 이메일 자동 수집**
  - **근거**: Step 3에서 사용자가 명시적으로 v1.0으로 미룸. 취업 메일 자동 수집은 MVP 범위 밖.

## v2.0+ (P2)

현재 요구사항 정의서에는 **개별 feature로 등재 안 함**. 범위 밖 항목은 다음과 같이 구분.

### v2.0+ 후보 (계획 상)
- 금융 데이터 자동 수집 (시장·포트폴리오 리밸런싱 신호)
- 웹 크롤링 자동 수집 (채용 공고·뉴스)
- 외부 변화에 플랜 자동 조정 (P3 문제의 고급 해결)
- 심화 대시보드·시각화
- 모바일 앱 클라이언트
- 다중 사용자 공유

### 명시적 v2.0+ 제외
- 범용 챗봇
- 공개 발행·블로깅
- 다중 사용자 공유·협업
- 장애인 접근성 확장 기능
- 웹 브라우저 UI (CLI·채팅 유지)
- 부재·누락 감지형 nudge (영구 금지 — 잔소리 방지 원칙)

## MVP 4주 Week별 포커스 (참고)

구현 순서는 design 단계에서 확정되지만, 기능 기준으로 대략:

| Week | 포커스 기능 |
|---|---|
| 1 | F-01~05 (캡처) + F-10~13 (검색) — 기반 |
| 2 | F-40~42 (세션) + F-50~52 (톤) — 대화 |
| 3 | F-20~23 (엔티티) + F-60~61 (자동화) — 상호작용 |
| 4 | F-30~34 (플래닝) + F-70~71 (연동) + F-80~82 (관리) — 완결 |

→ F-30 아침 브리핑이 Week 4에 완성됨으로써 MVP 최소 승리 조건 달성.

## Cut Line 결정 원칙 (참고용 메모)

- **MVP는 단일 성공 신호 중심**: 아침 브리핑 하나 완성도 낮으면 다른 걸 쌓아도 실패
- **잔소리 방지 우선**: 능동 감지 기능은 v1.0 이후로 미룸
- **데이터 필요한 학습 기능은 v1.0+**: 자동 승격·사용자 모델·챌린지
- **공존 원칙 유지**: 기존 시스템(OKR 트래커·태스크·캘린더) 대체 기능은 추가 안 함
